# WordFlip 单词卡片学习应用 — 产品需求文档

> 版本：v2.0（迭代版）  
> 整理日期：2026-06-28  
> 基础：Claude 与用户多轮讨论后确认的全部需求

---

## 1. 产品概述

**WordFlip** 是一款移动端单词卡片学习应用，核心体验围绕「翻转卡片」展开。通过小卡片展示单词，正面显示英文，背面显示中文释义，点击翻转。支持多本书籍合并学习、按组管理、艾宾浩斯复习、测验与统计。

---

## 2. 核心页面结构（7 个页面）

| 页面 | 说明 | 底部导航位置 |
|------|------|-------------|
| 今日（首页） | 今日任务、数据统计、开始学习入口 | 最右 |

## 3. 详细功能需求

### 3.1 今日页面（首页）

- **布局原则**：首页内容在手机容器内**无需滚动**即可全部展示。
- **顶部**：问候语 + 当前日期 + 连续打卡天数（火焰图标）。
- **中部**：三格数据统计卡片 — 已掌握单词数 / 待复习单词数 / 完成度百分比。
- **今日任务**：三个任务行 — 新词学习、到期复习、默写测验。每行显示所属词书与数量。
- **底部固定区域**：蓝色大按钮「开始学习」，固定在底部导航栏上方，点击后进入当前推荐组的卡片学习页。
- **底部导航栏**：从左到右依次为 设置 → 词书 → 分组 → 统计 → 今日（当前高亮在最右）。

### 3.2 单词书页面

- **仿设置页风格**：卡片式列表，清晰简洁。
- **选择词书**：支持多选，勾选/取消。相同单词在多书中去重，只出现一次。
- **显示每本词书进度**：已掌握数 / 总词汇数。
- **分组大小配置**：预设选项 10 / 20 / 30 / 50，也可自定义输入。选择后实时计算并显示「总组数」。
- **汇总信息**：底部显示已选总词数（去重后）、每组词数、总组数。
- **保存设置**：点「保存设置」确认，此配置长期有效。
- **手动添加分组**：单独入口，进入后从已选词书中点选任意单词组成自定义分组。

### 3.3 卡片学习页面（核心体验）

- **布局**：两列网格，每张卡片宽高比约 3:4.2。
- **卡片内容**：
  - **正面**：英文单词 + 词性 + 音标 + 「翻转」提示。
  - **背面**：中文释义 + 音标 + 「翻转」提示。
- **翻转动画**：
  - 使用 `cubic-bezier(0.34, 1.1, 0.64, 1)` 弹簧曲线，带回弹感。
  - 必须配合 `will-change: transform` 与 `preserve-3d`。
  - 翻转后必须手动再次点击才能翻回。
- **打乱动画**：
  - 点击「打乱」后，每张卡片先各自随机飞出去（方向、角度、距离均不同），然后重新随机飞回来落位。
  - 整个过程约 0.9 秒。
  - 打乱后保持各卡片独立翻转状态（已翻的仍翻，未翻的仍正）。
- **全翻按钮**：一键将所有卡片切换到背面（看中文）或全部翻回正面（看英文）。
- **发音**：翻到背面时，使用 **Web Speech API** 真实朗读单词发音（可在设置中关闭）。
- **长按详情（新增）**：
  - 长按任意卡片 → 底部抽屉（bottom sheet）弹出详情。
  - 展示内容：词义、例句 2-3 条、词根词缀拆解。
  - 抽屉有遮罩，点击遮罩或滑动可关闭。
  - 详情抽屉内提供「换一个污渍」和「关闭污渍」按钮。
- **首次引导（新增）**：用户第一次进入学习页时，显示一次性引导提示「长按卡片查看详细释义」，之后不再出现。
- **污渍记忆标记（新增）**：
  - 每张卡片的**正面**（英文面）可显示一个独一无二的「视觉污渍」作为记忆锚点。
  - 污渍类型：咖啡/茶渍、墨水污点、荧光笔划痕、折角/卷边、钢笔墨水渗透纹、蜡笔涂抹痕迹、橡皮擦残留屑等。
  - 每个单词绑定一个**固定污渍**（以单词 ID 为种子生成），永远不变。
  - 污渍透明度控制在 0.12–0.2 左右，位于卡片边缘或角落，不遮挡文字中心区域。
  - 分组详情的单词列表里可以预览并触发更换污渍（实际污渍只显示在卡片学习页）。

### 3.4 分组管理页面

- **分组列表**：展示各分组的进度条与状态标签（已完成 / 学习中 / 未开始）。
- **统计数据**：每组显示「已记得 / 模糊 / 未学 / 总词数」四个数字。
- **分组详情**：点击分组进入详情页，列出该组所有单词。
- **单词掌握状态**：每条单词旁可标记三档掌握度 — 记得（绿色）、模糊（橙色）、不认识（红色）。
  - 三档对应不同艾宾浩斯复习频率。
- **污渍入口**：在分组详情的单词列表中，每个单词条目旁提供入口，可查看/更换该单词的污渍（实际显示仍只在卡片学习页）。

### 3.5 默写测验页面

- **模式**：看中文释义，输入英文单词（纯输入，非四选一）。
- **交互**：
  - 输入框居中，输入后点「确认」或按回车提交。
  - 答对：输入框变绿色，显示「✓ 正确！」，自动标记该词为「记得」。
  - 答错：输入框变红色，显示「✗ 正确拼写：xxx」，自动标记为「模糊」并加入艾宾浩斯复习计划。
  - 自动进入下一题，间隔约 1.4 秒。
- **进度条**：顶部显示当前进度（第 N 题 / 总题数）与得分。
- **结果页**：
  - 显示表情、评价语（太棒了 / 不错 / 继续加油）。
  - 统计：答对数 / 答错数 / 正确率。
  - 提供「再来一次」和「返回」按钮。
- **Bug 修复**：所有进入测验的入口必须同时调用 `initQuiz()`，确保页面正确初始化。

### 3.6 统计 & 成就页面

- **学习数据四宫格**：已掌握单词 / 连续学习天数 / 测验正确率 / 累计学习天数。
- **热力日历**：近 3 个月，按每日学习量显示颜色深浅（4 级色阶）。
- **成就徽章**：列出可解锁成就（如坚持一周、千词达人、完成第一本词书等），显示已解锁 / 未解锁状态。

### 3.7 设置页面

- **学习体验**：
  - 翻转时自动发音（Toggle 开关）。
  - 艾宾浩斯间隔方案（标准 1-2-4-7-15-30 天）。
  - 默认学习方向（看英文回忆中文 / 看中文回忆英文）。
- **提醒**：
  - 每日学习提醒（可设时间，如 21:00）。
  - 复习到期提醒（有到期单词时额外通知）。
- **数据**：
  - 云端备份（显示上次备份时间）。
  - 导出单词本（CSV / Anki / Quizlet 格式）。

## 4. 全局交互规范

### 4.1 页面转场
- 底部导航切换：当前页向左滑出（`slide-left`），新页从右进入。
- 子页面（如学习页、详情页）：使用 `pushPage` / `popPage` 栈式导航，返回时从右滑出。
- 转场时长：0.32–0.35 秒，使用 `cubic-bezier(0.4, 0, 0.2, 1)`。

### 4.2 底部导航栏
- 高度 60px，白色背景，顶部 1px 分割线。
- 顺序（从左到右）：设置 → 词书 → 分组 → 统计 → 今日。
- 当前项高亮：图标放大 + 文字变为蓝色 + 加粗。

### 4.3 Toast 提示
- 位置：底部导航栏上方居中。
- 样式：黑色半透明圆角胶囊，白字，2 秒后自动消失。
- 用于：保存成功、标记状态、操作反馈等轻量提示。

## 5. 数据模型（示例）

### 5.1 单词结构
```json
{
  "en": "abandon",
  "cn": "放弃；抛弃",
  "pos": "v.",
  "ph": "/əˈbændən/",
  "detail": {
    "meaning": "to leave behind, to give up",
    "examples": [
      "He abandoned his car in the snow.",
      "The baby was abandoned on the church steps."
    ],
    "etymology": "a- (to) + bandon (control, authority)"
  }
}
```

### 5.2 分组结构
```json
{
  "id": 1,
  "name": "第1组",
  "total": 20,
  "ok": 8,
  "fuz": 3,
  "unknown": 2,
  "status": "learning"
}
```

### 5.3 艾宾浩斯复习间隔
标准方案：1 天 → 2 天 → 4 天 → 7 天 → 15 天 → 30 天。  
「模糊」单词按标准间隔复习；「不认识」单词缩短间隔；「记得」单词延长间隔。

## 6. 技术实现要点

- **翻转动画**：CSS `transform: rotateY(180deg)` + `transform-style: preserve-3d`。
- **打乱动画**：使用 CSS 自定义属性（`--tx`, `--ty`, `--tr`）配合 `@keyframes` 实现随机飞入飞出。
- **发音**：使用浏览器原生 `speechSynthesis.speak()`，无需外部网络请求。
- **污渍生成**：基于单词字符串做种子生成伪随机数，确保同一单词永远得到相同污渍；使用 SVG 或 CSS 伪元素绘制污渍，设置 `mix-blend-mode: multiply` 或低透明度融合。
- **本地存储**：使用 `localStorage` 保存用户设置、学习进度、污渍配置、首次引导标记等。
