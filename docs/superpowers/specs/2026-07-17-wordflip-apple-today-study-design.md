# WordFlip Apple 风格今日页与学习页重构设计

日期：2026-07-17
状态：已获用户口头批准，待实施
范围：Android `app`、`feature-today`、`feature-study` 与必要的 `core-ui`

## 1. 目标

在不改变服务端契约、学习业务规则与现有 v7/FSRS 数据迁移的前提下，重建 WordFlip 的今日页、学习页与五 Tab 导航壳。新体验采用 Apple 风格的层级、材质、排版、直接操控和可中断动效；不再遵循仓库现有 Natural Sage UI 规范，但仍使用 Android/Compose 原生能力实现。

本轮成功标准：

- 今日页打开后，用户能立即识别并进入最值得继续的学习分组。
- 学习页提供三种可持久化视图，且共享同一套卡片、详情、媒体和业务状态。
- 五 Tab 使用统一的浮动玻璃导航，其余四个 Tab 的页面内容不变。
- 翻卡、浏览、发音和学习 session 不得更新 FSRS、掌握度或测验结果。
- Light/Dark、大字体、系统安全区和减少动态效果可正常使用。

## 2. 已确认的产品决定

### 2.1 全局导航壳

- 保留现有五 Tab、顺序、路由与返回栈语义。
- 将 Material 3 默认底栏替换为浮动玻璃导航：圆角容器、半透明材质、轻边缘高光和克制阴影。
- 当前项使用系统蓝强调色和更高字重；未选项降低对比度，但仍满足可访问性要求。
- 导航壳仅出现在五个主 Tab；学习、测验、详情、卡拍等子页继续隐藏底栏。
- 设置、词书、分组、统计只继承新导航壳，不修改其页面内容。

### 2.2 今日页

采用“继续学习优先”结构：

1. 顶部展示本地问候、服务端日期、连续学习天数与提醒入口。
2. 主视觉卡优先使用 `recommendedStudy`；若为空则使用第一条 `recentGroups`；两者都为空时展示不可点击的无推荐状态，不虚构分组或数量。
3. 主卡点击进入学习；当主卡来自最近学习组时，保留进入该组测验的次操作。
4. 三项统计合并为一个紧凑分组：已掌握、待复习、完成度。
5. 今日任务使用单一分组列表展示新词、到期复习和默写测验，保留现有导航映射。
6. 最近学习仍最多展示三项，并保留“主点击学习、次操作测验”；可置于任务区之后，UI 顺序以本设计为准。
7. 已有内容刷新失败时保留旧数据并 Toast；首次加载使用贴合最终结构的骨架，首次失败原位展示重试。

今日页继续以服务端 `TodayDashboard` 为真相。客户端不计算任务数、掌握数、完成度、SRS 或推荐优先级。

### 2.3 学习页三种视图

学习页提供以下视图，并共享 `StudyUiState.Content.orderedWords`、`flipStates` 和现有媒体/详情能力：

1. **专注卡组**：一次显示一张大卡，横向拖动切换；相邻卡只提供轻微空间提示。
2. **自适应卡片墙**：手机默认两列，宽屏按最小卡宽自适应；适合快速扫描与批量翻面。
3. **焦点＋缩略轨道**：大卡承担主要学习，底部缩略轨道提供位置感和快速跳转。

默认值为“焦点＋缩略轨道”。用户从学习页右上角打开源点明确的视图菜单，选择结果写入学习模块的 DataStore key，并持久化最后一次选择。设置、词书、分组、统计页面内容保持不变。

视图切换只改变布局，不重排数据、不重置每卡翻转状态、不触发 session 上报。三种视图中同一卡片以稳定业务 key 关联状态；现阶段沿用已有 ViewModel 的 key，不在本轮处理 `wordKey` 到 `cardId` 的跨层迁移。

### 2.4 学习交互

- 轻点卡片切换正反面；只有正面翻到背面且自动发音开启时才触发 TTS。
- 长按约 500ms 且移动未超过 touch slop 时打开详情，并在提交长按的同一时刻触发一次触觉反馈。
- 专注卡组与混合视图的横向拖动按手指 1:1 跟随；松手后使用速度投影决定上一张、当前张或下一张。
- 常规布局与状态切换使用接近临界阻尼的 spring，不增加无来源弹跳；只有带速度的拖动结束可使用轻微回弹。
- 所有按压目标在按下时立即给出约 `0.97` 的缩放反馈；动画不能阻塞点击、返回或新手势。
- 翻卡保留 3D 透视，并从当前屏幕角度重新定向，支持快速反向操作。
- 打乱、全部翻面、详情、图片、污渍、TTS 和进入测验的功能保留。打乱后的卡片翻转状态保持不变。
- 学习页不添加“记得、模糊、不认识”等掌握度按钮。

### 2.5 详情与叠层

- 保留当前返回优先级：选图 Sheet → 图片编辑器 → 详情 → 离开学习页。
- 详情继续展示当前学习卡考义、全部义项、来源资料、例句、发音、污渍和图片操作。
- 详情改为更清晰的 Apple 风格底部材质层；从触发卡片所在空间方向出现和返回，不改变数据接口。
- 图片编辑器保持全屏；打开时隐藏学习页顶栏，Toast 仍位于编辑器之上。
- 本轮不恢复已经移除的客户端词典切换逻辑。

## 3. 视觉系统

### 3.1 色彩与材质

- 使用中性灰背景、白色/深灰表面和系统蓝强调色；不继续使用 Natural Sage 作为这两页与导航壳的主视觉。
- 玻璃材质只用于浮动导航、菜单和叠层等功能层；内容卡片使用较实的表面，避免玻璃叠玻璃。
- Light/Dark 分别提供独立颜色值，不能简单反相。
- 大面积表面的模糊、透明度和阴影强于小控件，但保持克制，避免影响文本清晰度。

### 3.2 排版与间距

- 继续使用系统字体，依赖平台字形与用户字体缩放。
- 今日页问候使用大标题和紧凑字距；正文保持舒适行高；小标签增加少量字距。
- 所有可点击区域至少 48dp；图标与文本按视觉中心而非几何边界对齐。
- 卡片和导航优先使用 22–30dp 圆角；小型胶囊只用于状态、计数或紧凑操作。

## 4. 组件与代码边界

预计新增或调整的职责：

- `app/navigation`：新增浮动 Tab 导航 Composable；保留现有导航事件与 route。
- `feature-today`：重写页面展示组件；保留 `TodayViewModel`、Repository 与导航回调。
- `feature-study`：新增视图模式模型、偏好读取和三种布局；尽量不修改现有 ViewModel 的业务行为。
- `core-ui`：仅放跨 feature 复用的玻璃表面、按压反馈或 Apple 风格基础组件；不得依赖 feature 模块。

当前工作树包含大量未提交的 v7/FSRS 改动。实施必须使用最小差异并逐文件核对，禁止覆盖 `StudyViewModel`、DTO、Repository、OpenAPI 或服务端迁移中的无关改动。

## 5. 数据流

### 5.1 今日页

`TodayRepository` → `TodayViewModel.uiState` → 新 Today Composable。页面只格式化日期、问候和相对时间；所有统计与任务数量直接展示服务端值。导航仍通过既有回调进入学习或创建新的测验 session。

### 5.2 学习页

`StudyRepository` → `StudyViewModel.uiState` → 当前视图模式对应的布局。视图模式来自 DataStore，并作为纯展示偏好参与组合；卡片翻转、打乱、详情和媒体操作继续调用现有 ViewModel 方法。

离开学习页时继续沿用现有 session 上报流程。本轮不改变顶栏返回、系统返回或进入测验时的业务触发时机；session 上报仍不得写入 FSRS。

## 6. 错误、空态与降级

- 今日首次失败：保留页面骨架层级，在内容区显示错误与重试。
- 今日静默刷新失败：保留旧内容并 Toast，不闪回 Loading。
- 无推荐学习：主卡显示“暂无推荐学习”，任务列表仍可操作。
- 学习组为空：显示明确空态和返回操作，不渲染空 pager/grid。
- 学习数据失败：显示原位重试，不丢失返回能力。
- 图片、污渍或 TTS 失败：沿用一次性 Toast，不伪造成功状态。
- 减少动态效果：位置动画和弹簧降级为短交叉淡入或立即切换；保留颜色、透明度等非前庭反馈。
- 设备不支持高质量模糊时：使用较高不透明度的实色表面和边缘高光，保证对比度。

## 7. 测试与验收

### 7.1 自动验证

- 为学习视图模式的序列化、默认值和持久化映射添加单元测试。
- 为今日主卡数据选择策略添加纯函数测试：推荐学习优先、最近组回退、无数据空态。
- 为仅“正面 → 背面”触发自动发音的判断添加单元测试或最小可测逻辑。
- 编译 `feature-today` 与 `feature-study`。
- 运行 Android 单测、组装 Debug APK，并执行 `git diff --check`。

### 7.2 手工验收

- 五 Tab 路由、选中状态、返回栈和子页隐藏规则不变。
- 今日主卡、任务和最近组的学习/测验入口映射正确。
- 三种学习视图可即时切换、持久化并在重启后恢复。
- 切换视图不重置翻转状态；打乱后也保留翻转状态。
- 翻卡、长按、横向拖动、详情返回、图片编辑器返回不存在手势冲突。
- Light/Dark、字体放大、窄屏、宽屏、导航栏安全区和减少动态效果正常。
- 仅浏览或翻卡后，FSRS/掌握度不变；完成有效测验后才发生服务端记忆更新。

## 8. 非目标

- 不重构设置、词书、分组、统计四页内容。
- 不修改 OpenAPI、服务端 DTO、FSRS 算法或测验判题。
- 不完成 `wordKey` 到 `cardId` 的剩余迁移。
- 不新增第三方动画或毛玻璃依赖，优先使用 Compose Foundation、Animation 和 Graphics API。
- 不在本轮提交 Git commit、push 或 PR，除非用户另行明确要求。
