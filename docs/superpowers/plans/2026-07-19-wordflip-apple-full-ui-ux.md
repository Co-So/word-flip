# WordFlip 全应用 Apple UI/UX 重构实施计划

> **执行方式：** 每个任务由独立实现代理串行完成，随后由另一独立代理只读审查；不得提交、暂存、还原或覆盖无关改动。

**目标：** 在不改变服务端权威业务规则的前提下，为 Android 应用所有现有页面建立统一的 Apple 风格视觉和更清晰的操作路径。

**架构：** `core-ui` 提供统一的 Apple 语义色和页面原语；各 feature 仅替换 Compose 呈现层，继续复用现有 ViewModel、Repository、回调和路由。学习朗读修复先由单元测试锁定，翻转浏览不写入 FSRS 或掌握度。

**技术栈：** Kotlin、Jetpack Compose、Material 3、Navigation Compose 2.8.4、Hilt。

## 全局约束

- 不修改 OpenAPI、DTO、服务端接口、ViewModel 数据职责或路由参数。
- 测验是唯一可写入 FSRS/记忆状态的入口；学习翻转、朗读、图片和污渍操作不得改掌握度。
- 保留当前未提交的 v7/FSRS 改动；不创建 worktree，不自动提交或推送。
- 所有新增或修改的业务代码使用简体中文注释；图标使用 Material Symbols。
- 只使用现有 Compose/Material 3 依赖；轻量动效只表达导航、抽屉和操作反馈，高频学习操作不增加延迟。

## 任务

### Task 8：Apple 基础视觉与浮动导航壳

- 统一浅/深色 Apple 语义色，将现有页面可复用地接入页面标题、分组卡片、信息卡、主按钮、操作行、空态和安全区。
- 保留五 Tab、状态恢复、子页隐藏底栏与现有路由；选中项使用低饱和胶囊底。
- 今日页仅做令牌、安全区与导航一致性整理。

### Task 9：学习朗读与详情抽屉

- 先将 `StudyInteractionTest` 改为：自动朗读开启时正反面和连续两次翻转都朗读，关闭时均不朗读；再让 `shouldAutoSpeakAfterFlip` 仅返回开关状态。
- 重构详情抽屉为词形信息、关闭/朗读主操作、语速步进器、本书考义、可展开来源资料和记忆素材分组。
- 保留所有来源、污渍、媒体、TTS、返回和状态恢复回调。

### Task 10：首次体验、认证、设置与统计

- PlanGate、认证页、设置页和统计页统一为 Apple 分组表面与清晰单一主操作。
- 设置中的自动朗读文案改为“点击学习卡片时自动发音（正反面）”；原有 DataStore/PATCH、TTS 提示和占位能力语义不变。
- 统计补齐热力和成就的合并无障碍描述。

### Task 11：词书、分组与学习入口

- 重组词书、词书详情、导入预览、分组、分组详情、手动分组与污渍模式的布局和主次操作。
- 保留词书切换、导入确认、搜索筛选、仅未入组卡可手动分组、热力只读和现有导航回调。

### Task 12：测验、卡拍与图片编辑

- 将测验改为单焦点题干和固定答题区；将结果、错误巩固和选择题层级变得更清晰。
- 重组卡拍操作表及图片编辑器工具区；保留权限、临时 URI、上传/PATCH、图片显示中文和返回优先级。

### Task 13：全量验证与终审

- 运行模块测试、全量 `test`、`assembleDebug` 和 `git diff --check`。
- 独立终审 UI/UX、无障碍与业务禁区；使用设备或模拟器完成浅深色、字号、IME、导航、测验与媒体手工验收。
